# Medibox 测试计划 (Test Plan)

**版本**: 1.0
**状态**: 待执行
**最后更新**: 2024-05-20

---

## 📌 测试目标

本测试计划旨在验证 Medibox 智能药箱系统的整体质量，确保其满足阿尔兹海默症患者及其护理人员的特殊需求。

1.  **功能验证**: 确保所有核心功能（提醒、开盖检测、语音播报）按预期工作。
2.  **硬件可靠性**: 验证 Pogo Pin 连接器和传感器在长期使用中的稳定性（模拟 10,000 次插拔/开关）。
3.  **用户体验**: 确保 60+ 老年用户能理解语音提示并顺利完成服药流程。
4.  **系统性能**: 保证 I²C 通信和云端同步的实时性。

---

## 🔬 测试范围

### 1. 单元测试 (Unit Testing)
| 模块 | 测试内容 | 负责人 |
|------|----------|--------|
| **ESP32 Firmware** | RTC 时间同步准确性<br>I²C Master 指令发送<br>TTS 语音合成模块初始化 | Embedded Dev |
| **ATtiny85 Firmware** | 霍尔传感器状态读取<br>LED 呼吸灯控制逻辑<br>I²C Slave 地址响应 | Embedded Dev |
| **Frontend UI** | Grid 组件渲染逻辑<br>PillboxCard 状态更新<br>ReminderForm 表单验证 | Frontend Dev |

### 2. 集成测试 (Integration Testing)
- **I²C 主从通信**: 验证 Hub (ESP32) 与多个 Slave (ATtiny85) 之间的数据包完整性，特别是地址冲突处理。
- **WebSocket 实时同步**: 验证前端操作（如“Identify”）是否能在 <200ms 内触发硬件 LED 闪烁。
- **TTS 链路**: 验证从“提醒触发”到“语音播报”的完整调用链路。

### 3. 系统测试 (System Testing)
- **端到端流程 (E2E)**:
  1.  Web 设置 08:00 服药提醒。
  2.  08:00 到达，Hub 播放语音，对应 Slave 闪烁。
  3.  用户打开药盒，检测到 `OPEN` 事件。
  4.  Hub 停止提醒，记录服药时间。
  5.  Web 界面更新为“已服药”。
- **多药盒并行**: 挂载 3-5 个从机模块，验证系统负载和稳定性。
- **异常恢复**:
  - 断网重连测试。
  - 断电重启后 RTC 时间与状态恢复。

### 4. 性能测试 (Performance Testing)
- **响应时间**: 霍尔传感器触发到语音反馈 < 200ms。
- **通信延迟**: I²C 轮询周期 < 100ms。
- **功耗**: 待机功耗 < 2W，峰值功耗 < 20W。

### 5. 可靠性测试 (Reliability Testing)
- **寿命测试**: Pogo Pin 插拔循环 1,000 次，触点无接触不良。
- **稳定性**: 连续运行 72 小时无死机、无内存泄漏。
- **老化测试**: 模拟高温/高湿环境下的长期运行（可选）。

### 6. 用户验收测试 (UAT)
- **目标用户**: 5 位 60-80 岁的老年人及其家属。
- **测试场景**:
  - 独立完成一次服药流程。
  - 家属完成网络配置和药盒映射。
  - 语音清晰度主观评分 (1-5 分)。

## 📝 测试用例 (Test Cases)

### TC-001: I²C 主从通信稳定性
- **前置条件**: 连接 1 个 Hub 和 3 个 Slaves。
- **操作步骤**:
  1. Hub 每秒向所有 Slave 发送状态查询指令。
  2. 持续运行 1 小时。
- **预期结果**: 无丢包，无 I²C Bus 挂死，所有 Slave 正常响应。

### TC-002: 霍尔传感器开盖检测
- **前置条件**: 药盒处于关闭状态。
- **操作步骤**:
  1. 打开药盒盖子。
  2. 保持 5 秒。
  3. 关闭药盒盖子。
- **预期结果**:
  - 开启瞬间，Slave LED 状态改变。
  - Hub 收到 `OPEN` 事件。
  - 关闭瞬间，Hub 收到 `CLOSE` 事件。

### TC-003: TTS 语音播报
- **前置条件**: 系统音量设置为 50%。
- **操作步骤**: 触发“服药提醒”。
- **预期结果**: 语音清晰播报：“现在是服药时间，请服用降压药 2 粒。”

### TC-004: 5 分钟重复提醒
- **前置条件**: 设置提醒但未服药。
- **操作步骤**:
  1. 提醒触发后，不打开药盒。
  2. 等待 5 分钟。
- **预期结果**: 5 分钟后再次播放语音提醒，直到药盒被打开。

### TC-005: Blink to Identify 映射
- **前置条件**: Web 界面已连接。
- **操作步骤**: 在 Web 点击“Identify”按钮。
- **预期结果**: 目标物理药盒蓝色 LED 快速闪烁 5 秒，其他药盒无反应。
